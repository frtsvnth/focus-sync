# Промпт для агента-кодера: Синк Agenda

## Задача
Создай веб-сервис **«Синк Agenda»** — генератор красивых названий для повестки командного синка.
Сервис должен быть чисто фронтальным (HTML + CSS + JS в одном файле `index.html`),
готовым к деплою на **GitHub Pages** без какого-либо бэкенда.

***

## Технологии

- Чистый HTML5 / CSS3 / Vanilla JS — никаких фреймворков, никакого Node.js
- Шрифты: Cabinet Grotesk (заголовки) + Satoshi (основной текст) через Fontshare CDN
- AI API: **GigaChat API** (Сбер) — российский LLM, доступен напрямую
  - OAuth endpoint (server-only): `https://ngw.devices.sberbank.ru:9443/api/v2/oauth`
  - Chat endpoint: `https://gigachat.devices.sberbank.ru/api/v1/chat/completions`
  - Модель: `GigaChat`
  - **⚠️ OAuth не поддерживает CORS** → нужен Cloudflare Worker как прокси
  - Authorization Key (Base64): `MDE5YWZjYTgtODkwYy03NWMyLWE3NTMtZmVhZDZkY2E2ZmFlOjVkNGJjZWVjLTNkMWMtNGJkYy04MzU2LTQyODZmMjlhMmZmZg==`
  - Scope: `GIGACHAT_API_PERS`
- Праздники: **Nager.Date API** — `https://date.nager.at/api/v3/PublicHolidays/{year}/RU`
  (бесплатный, без ключа, поддерживает CORS)

## Логика работы

### 1. Инициализация
- При загрузке страницы автоматически запускается генерация **на завтра**
- Вычислять `_today` и `_tomorrow` через `new Date()` с обнулением времени

### 2. Сбор праздников (async)
1. Параллельный запрос к `https://date.nager.at/api/v3/PublicHolidays/{year}/RU`
   - Фильтровать по `h.date === fmtDate(targetDate)` (формат YYYY-MM-DD)
   - Брать `h.localName || h.name`
   - Таймаут: `AbortSignal.timeout(5000)`
2. Встроенная база праздников по ключу `MM-DD` (около 60 записей)
   - Примеры: `'07-08': ['💑 День семьи, любви и верности']`, `'03-14': ['🥧 День числа Пи (3.14)']`
3. «Причудливый» праздник — массив QUIRKY из ~24 строк, индекс = `(year*10000 + month*100 + day) % QUIRKY.length`
   - **Важно**: индекс определяется датой, а НЕ днём недели! Нет слова "Пятница" в строках
4. Объединить все в Set, если пустой — добавить QUIRKY, в конце всегда добавить QUIRKY как доп. кандидат

### 3. AI-запрос (нормальная генерация)
Промпт (строгий формат):
```
Ты генератор названий для повестки командного синка.
Ответ — СТРОГО ОДНА строка в формате: 📆 {DD.MM} <Название праздника или темы>
Никаких кавычек, пояснений, лишних символов — только эта строка.

Список праздников и тем на {DD.MM}:
• {holiday1}
• {holiday2}
...

Выбери из списка одно — самое интересное, необычное или смешное.
Используй эмодзи из списка (не добавляй новые).
Пример правильного ответа: 📆 {DD.MM} День числа Пи и бесконечных задач
```

Санитизация ответа: если в ответе нет `DD.MM` строки — собрать fallback из локального списка.

### 4. AI-запрос (Дичь)
Промпт тоже требует строгий формат `📆 DD.MM ...`:
```
Придумай ОДНО максимально дикое, абсурдное и смешное название для повестки командного синка.
Ответ — СТРОГО ОДНА строка: 📆 {DD.MM} <дикое название>
Не более 10 слов после даты. Без кавычек, пояснений. Только эта строка.
Пример: 📆 {DD.MM} Срочный консилиум по поводу пропавшего дедлайна
```

Fallback-массив (7 строк без даты — к ним надо дописать `📆 DD.MM ` в коде если AI не ответил).

### 5. Ограничения (в памяти, без localStorage)
- `genUsed: false` — использован ли лимит на обычную генерацию **сегодня**
- `madUsed: false` — использован ли лимит на "Дичь"
- `unlocked: false` — снят ли пин-ограничение
- **Пин: "112"** — 3 поля по 1 цифре; при вводе 3-й цифры — автопроверка
- При `unlocked = true`: кнопка "Ещё раз" и "Дичь" работают без ограничений
- **При смене даты (сегодня/завтра/другой)**: `genUsed` сбрасывается в `false` → новая генерация

### 6. UX деталей
- **Лоадер**: виден пока идут запросы. Показывает два состояния: "Ищу праздники..." → "Выбираю самый интересный..."
- **Эффект печатной машинки**: текст в `res-box` появляется посимвольно (24ms + random 14ms)
  - Живой мигающий курсор во время печати; удаляется после завершения
- **Кнопка Скопировать**: при клике меняет иконку и текст на "✓ Скопировано", сбрасывается через 2с
- **Кнопка ключ**: при наличии ключа — зелёная с галочкой `Ключ ✓`
- **Лимит бейдж**: при `unlocked` — зелёный с `∞ Разблокировано`
- **Тост**: фиксированный снизу по центру, исчезает через 3с; варианты `.suc` (зелёный) и `.err` (красный)

## Модальные окна

### Модал "Ключ" — УБРАН
Ключ захардкожен в Cloudflare Worker Secrets — пользователю вводить ничего не нужно.
В `index.html` есть только константа `PROXY_URL` — URL задеплоенного воркера.

### Модал "Пин" (при клике "разблокировать" или "Ещё раз" когда лимит)
- Три input[type=password] по 1 цифре (автофокус следующего при вводе, backspace — назад)
- При вводе 3-й цифры автоматически проверяет
- Анимация тряски при неверном пине
- Пин: **112**

### Анимация модалов
- Overlay: `opacity: 0 → 1`, backdrop-filter: blur(10px)
- Modal: `scale(0.95) → scale(1)`, transition cubic-bezier(.16,1,.3,1)

## Скрытие API-ключа (опциональный Cloudflare Worker)

**Правда жизни**: в чистом frontend-приложении на GitHub Pages невозможно технически спрятать ключ от опытного разработчика через DevTools.

**GigaChat OAuth не работает из браузера** (нет CORS на `ngw.devices.sberbank.ru:9443`).
Cloudflare Worker — **обязателен**, не опционален.

**Архитектура:**
```
Браузер → Cloudflare Worker → [OAuth → получить токен] → GigaChat API
```

**Код воркера** — см. файл `cloudflare-worker-gigachat.js`.

**Деплой:**
1. https://dash.cloudflare.com → Workers & Pages → Create Worker
2. Вставь код из `cloudflare-worker-gigachat.js`
3. Settings → Variables → Secrets:
   `GIGACHAT_AUTH_KEY = MDE5YWZjYTgtODkwYy03NWMyLWE3NTMtZmVhZDZkY2E2ZmFlOjVkNGJjZWVjLTNkMWMtNGJkYy04MzU2LTQyODZmMjlhMmZmZg==`
4. Скопируй URL воркера
5. В `index.html` замени `PROXY_URL` на URL воркера

**Что делает воркер:**
- Получает Access Token через OAuth (кешируется на 29 мин)
- Подставляет `Authorization: Bearer <token>` к запросам в GigaChat
- Возвращает ответ с CORS-заголовками

Бесплатный план Workers: 100,000 запросов/день.

## Важные антибаги

1. **Квёркевый праздник** — индекс по формуле `(year*10000 + month*100 + day) % QUIRKY.length`, **не** по дню недели
2. **Результат Дичи** — всегда начинается с `📆 DD.MM` (санитизация: если AI не вернул дату — добавить руками)
3. **Смена даты** — сбрасывает `genUsed = false`, чтобы позволить новую генерацию
4. **Нет localStorage** — всё состояние только в JS-переменной `const S = {...}`; при перезагрузке всё сбрасывается
5. **Фокус в PIN-модале** — `setTimeout(() => p0.focus(), 300)` чтобы анимация overlay успела завершиться
6. **Эффект печатной машинки** — вставлять символы через `insertBefore(textNode, cursorSpan)`, курсор удалить в конце
